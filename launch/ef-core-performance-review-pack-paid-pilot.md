## What this is

The pack is a downloadable ZIP with:

- buyer quickstart;
- sanitized evidence fixture;
- offline evidence scorer;
- generated SQL evidence table template;
- Query Store / runtime correlation guidance;
- migration-risk preflight snippets;
- report template and scoring rubric;
- checksum for local verification.

It is designed for local, redacted evidence. It does not need repository access, database credentials, remote access, production logs, or customer-data upload.

## Why it exists

EF Core performance discussions often start too late and too vaguely:

- “EF is slow.”
- “Maybe this query needs an index.”
- “This migration should be fine.”

The useful first step is usually narrower: capture generated SQL, record provider/version context, add runtime evidence when available, and separate local proof from production-change decisions.

This kit packages that evidence workflow so a team can run it internally before proposing query, index, or migration changes.
